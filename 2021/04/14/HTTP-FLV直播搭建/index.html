# RTMP 直播协议

  • 兼容性不高
    • 无法直接使用 html 的 video 标签播放：video.js，video-flash
  • 有延迟

RTMP 是专为流媒体开发的协议，对底层的优化比其它协议更加优秀

优点

  • 它对 Adobe Flash 支持好
  • 基本上所有的编码器（摄像头之类）都支持 RTMP 输出
  • 另外 RTMP 适合长时间播放，即 10 天多连续播放没有出现问题，稳定
  • 最后 RTMP 的延迟相对较低，一般延时在 1-3s 之间，一般的视频会议，互动式直播，完全是够用的

缺点

  • 它是基于 TCP 传输，非公共端口，可能会被防火墙阻拦，1935
  • RTMP 为 Adobe 私有协议，很多设备无法播放，兼容性不高
  • Html 基础支持不好，无法直接使用 HTML 标签完成拉流

# HTTP-FLV 直播协议

  • Youtube，B 站

流媒体数据封装成 FLV 格式

FLV: 简单轻量、加载速度极快、播放快、

  1. 推流，服务器把视频封装为.flv 一段段
  2. 浏览器拉流，下载视频
  3. Content-Type: rtmp/flv

优点

  • 可以直接通过 http 协议进行传输，http 属于公共协议，公共端口：80
    • html 播放视频就很简单， video
  • 支持加密，使用 https
  • 兼容移动端设备，比如：安卓、ios

缺点

  • 会缓存直播数据，保存直播传输过来的一段段 flv 文件，隐私保密不好
  • 不能在大并发情况下使用

Content-Type？这是啥？

  • image/jpeg、image/png

浏览器渲染数据的标准 Content-Type 格式，多用途邮件扩展协议，MIME ？用来做一套关于浏览器渲染数据的标准

# HLS 直播协议

  • 苹果公司的
  • 对于苹果系列产品支持优秀
  • 兼容性 ok，通过 HTTP 传输
  • 延迟极高，60s 都是可以接收的
  • ts 切片较小，会造成海量小文件，保存麻烦，缓存麻烦
